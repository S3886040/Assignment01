Project Idea

Overview

Have you ever wondered if the mail has arrived yet? Have you ever gone out to check the mail, only to find that there is nothing there? Well, my project idea aims to solve this problem. When the mail arrives, you will receive a notification on your mobile that “you’ve got mail”, hopefully in that very 90’s computer voice made famous by Meg Ryan and Tom hanks’ film You’ve got mail. The invention includes an app, downloadable on both Android and iPhone, which will notify you when the mail arrives and log the date and time it arrived.

Motivation

There are products on the market like this. This is not a new idea or invention. But my adaptation is a learning project making use of some great technologies and integrating them in a new way. And some other products will require connection to a smart hub, or they will relay your data/mail notifications through private server infrastructure then send out notifications with their apps. My solution will be directly connected to our home wi-fi network, and we will be programming what is sent to the cloud, so we can be sure what is being stored in remote databases.

Description

The mail detector will utilise several pieces of hardware working together to make this work. We will have an Arduino Nano 33 IoT, a magnetic door sensor and USB power supply. We will place the hardware into the letterbox placing one half of the door sensor on any movable part of the letter box and other half fixed to a non-moveable piece. If there are no moveable parts, a plate can placed at the back of the chute with hinges which will open when pressure is creating by the delivery of the daily mail. The Arduino Nano and power supply will remain inside the, hopefully locked, letter box and connected to your Wi-Fi. The basic function of the device is when the mail door cute is opened the magnetic door sensor ‘seal’ will be broken we will then program the Arduino to update the cloud that the mail has arrived creating an event stored in their servers. This event will then be available for the mobile app to access.

The second part is the mobile app we create, to check to see if the mail has arrived. There is the basic function of the app which would/could be a button that says, ‘Check Mail’. And when you click the button, an API is sent to the server to see if the mail has arrived today. But there are some layers we can add to this.

The first layer we can add is a history of all the dates and times the mail arrives. We could simply add a button which will then render a screen showing all, or at least the last 30 days of mail times and dates of arrival.

The second layer could be a notification feature which would notify the user when the mail arrives. Alerting them so they can go and retrieve it.

Tools and Technologies

As specified above we will have the Arduino Nano. Which was chosen as an appropriate piece of hardware because of its simplicity but also because of its capability. Namely its wi-fi capability. Using this we can connect directly to a server to store our events. And this creates a much larger scope for this project as we can then connect devices any where in the world. We can in theory, check the mail while in Japan. Without independent wi-fi connection we would be limited to local connection on our own network. And Arduinos cloud system offers a simple cloud storage solution with API integration. And is also free to get started.

We will also need a magnetic door seal, and power supply. A generic brand electronic door seal would be appropriate, and any USB power supply will work. However, the longer the battery life the better.

Software wise, we will need the Arduino IDE to develop the code which will send the events and relevant details to the cloud.

And to develop the mobile app which the user will interact with. We could use React Native. Which would then make it a cross platform application, being compatible on both Android and iPhone. This would be extra useful if this ever became commercially viable.

Skills Required

Firstly, we would need a solid understanding of the Arduino coding library. This is their custom code library they designed for ease of use to function along side their hardware and is integrated in into the Arduino Integrated Development Environment (IDE). And we would need to learn how this library is compatible with API integration. Next, we would need to learn React Native, so we could code up the user facing app.

These skills are very attainable. The Arduino coding library is designed for ease of use and for beginners to learn coding alongside hardware integration. So, this would be a learning curve however very doable. React Native, on the other hand, is a bit more complex and would require a lot of time to learn. But our app is quite basic and the functionality beyond the API integration is quite simple.

But also, since this is a build project, we would have all the knowledge we gained from building this from start to finish.
